# 副業アイデア比較

複数の副業アイデアを比較マトリクスで並べ、ユーザーの状況に最適なアイデアを推薦する。

## 入力

比較対象のアイデア（2〜5個）。$ARGUMENTS で親スキルから評価済みアイデアが渡された場合はそれを使用する。

## 比較マトリクス

以下の形式で比較表を作成する:

| 比較項目 | アイデアA | アイデアB | アイデアC |
|----------|-----------|-----------|-----------|
| 初期投資 | | | |
| 月収ポテンシャル | | | |
| 初収益までの期間 | | | |
| 必要時間/週 | | | |
| スケーラビリティ | | | |
| リスク | | | |
| スキル適合度 | | | |
| **4条件スコア** | | | |
| 希少性 | ◎/○/△/× | ◎/○/△/× | ◎/○/△/× |
| ニーズ | ◎/○/△/× | ◎/○/△/× | ◎/○/△/× |
| 潜在性 | ◎/○/△/× | ◎/○/△/× | ◎/○/△/× |
| 参入障壁 | ◎/○/△/× | ◎/○/△/× | ◎/○/△/× |
| **7問スコア** | X/7 YES | X/7 YES | X/7 YES |
| 総合スコア（★） | | | |

## 優先事項に基づくランキング

ユーザーの優先事項（以下の中から確認）に応じてランキングを変える:

- **即金性重視**: 初収益までの期間を最重要視
- **高収入志向**: 月収ポテンシャルを最重要視
- **低リスク志向**: リスクの低さとスキル適合度を重視
- **スケーラビリティ重視**: 仕組み化可能性を最重要視
- **最小時間**: 必要時間の少なさを重視
- **4条件重視**: 4条件の加重スコアを最重要視

### 4条件加重スコア計算

◎=4点、○=3点、△=2点、×=1点として、以下の重みで加重平均を算出:

| 条件 | 重み | 理由 |
|------|------|------|
| 希少性 | 25% | 差別化は持続的な競争力の源泉 |
| ニーズ | 30% | 需要がなければビジネスは成立しない |
| 潜在性 | 25% | 他が気づいていないことが先行者優位を生む |
| 参入障壁 | 20% | 初期は弱くても時間で構築可能 |

## 推薦コメント

比較マトリクスとランキングに基づき:

- **最もおすすめ**: 総合的に最適なアイデアとその理由
- **堅実な選択肢**: リスクが低く始めやすいアイデア
- **高リターン候補**: リスクを取れるなら最も成長性が高いアイデア
- **Blue Ocean Champion**: 4条件スコアが最も高いアイデア

## 組み合わせ提案

可能な場合、複数のアイデアを組み合わせる戦略も提案する:
- 短期（即金性の高いアイデア）+ 長期（ストック型のアイデア）
- 相乗効果（一方の作業が他方のコンテンツになる等）
- 時間配分の提案（週X時間をA、Y時間をB）

## validator未実施の場合

競合検証（side-biz-validator）がまだ実行されていない場合、4条件スコアは「暫定」と明記し、比較結果の確度が低い旨を警告する。最終決定前にvalidatorの実行を推奨する。
