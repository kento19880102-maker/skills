# ヒアリング出力テンプレート

## プロファイルテーブル

ヒアリング完了後、以下の形式で整理する。

| 項目 | 内容 |
|------|------|
| スキル・経験 | （ヒアリング結果） |
| 興味・関心 | （ヒアリング結果） |
| 可処分時間 | （ヒアリング結果） |
| 初期予算 | （ヒアリング結果） |
| リスク許容度 | 低（確実に月1-3万）/ 中（半年後に月5-10万）/ 高（1年後に月30万以上） |
| 有料サービス | （月額合計・主要サービス一覧） |
| 無料サービス | （毎日/毎週使用するアプリ・サービス） |
| 不満・困りごと | （最近イライラしたこと・不便だと感じたこと） |
| 周囲の不満 | （家族・同僚・友人の愚痴・不満） |
| 願望・変化 | （将来の夢・目標、最近の大きな変化） |

## ユニーク特性分析

全10項目を横断分析し、「この人だけの交差点」を2-3個特定する。

### 分析の観点

- どの属性の組み合わせが希少か？（例: 物流ワーカー × AI活用 × 科学的自己改善）
- 有料サービスの利用パターンから見える独自の関心領域は？
- 不満と願望の交差点にビジネスチャンスはないか？
- 周囲の不満の中で、ユーザーのスキルで解決できるものはないか？

### 出力形式

```
### ユニーク特性
1. [交差点1]: （どのスキル×どの関心が掛け合わさっているか、なぜ希少か）
2. [交差点2]: （同上）
3. [交差点3]: （同上）
```

## ヒアリングの会話のコツ

- 質問は具体的に。「何ができますか？」より「本業ではどんな仕事をしていますか？」
- ユーザーが曖昧な回答をした場合は、選択肢を提示して選んでもらう
- Part 2はカジュアルな導入で。「ちなみに」「ついでに聞くけど」等
- 全10項目が揃わなくても、最低限「スキル」と「可処分時間」＋Part 2から2項目以上あれば次に進んでよい
