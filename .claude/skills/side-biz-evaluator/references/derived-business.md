## Tier定義

副業を継続することで蓄積されるスキルから、段階的に新しいビジネスに展開できる。

### Tier 1: 自然な延長線（6-12ヶ月後）

- 本業の副業で培ったスキルを**そのまま**別の形で提供
- 追加の学習コストが低い
- 既存のオーディエンス・データを直接活用可能
- **例**: コンテンツ制作 → テンプレート販売、DB運営 → コンサルティング

### Tier 2: スキル深化後（12-18ヶ月後）

- 本業の副業で培ったスキルを**深化・組み合わせ**て提供
- 一定の追加学習が必要
- 新しいオーディエンスにリーチ可能
- **例**: ブログ運営 → オンライン講座、データ分析 → マイクロSaaS

### Tier 3: プラットフォーム化（18ヶ月以降）

- 本業の副業を**プラットフォーム/エコシステム**に拡張
- 大きな投資（時間 or 資金）が必要
- ネットワーク効果・データモートが発生
- M&A/買収の対象になりうる
- **例**: DB → ユーザー投稿型プラットフォーム、講座 → 教育プラットフォーム

## スキル獲得テーブル

派生ビジネス分析では、以下の形式でスキル獲得を整理する:

| # | スキル | 主にどの活動から | 習得期間 | Tier 1派生 | Tier 2派生 |
|---|--------|------------------|----------|------------|------------|
| 1 | （例: SEOライティング） | （例: ブログ記事執筆） | （例: 1-3ヶ月） | （例: SEOライター受託） | （例: SEO講座） |

### よく獲得されるスキルカテゴリ

- **コンテンツ制作系**: SEOライティング、動画編集、SNSマーケティング、ストーリーテリング
- **テクニカル系**: Webサイト構築、データベース設計、AI活用、ノーコードツール
- **ビジネス系**: アフィリエイト設計、価格設定、顧客獲得、コミュニティ運営
- **分析系**: サービス比較分析、データ可視化、市場調査
- **ブランド系**: パーソナルブランディング、信頼構築、ネットワーキング

## 代表的な派生パス

### コンテンツ制作 → スケール

```
ブログ/note記事 → 有料マガジン → オンライン講座 → コミュニティ → 書籍化
```

### データベース → プラットフォーム

```
個人運営DB → SEOトラフィック → ユーザー投稿機能 → プラットフォーム化 → M&A
```

### 実験/検証 → 権威性

```
n=1実験記録 → データ蓄積 → 専門家ポジション → コンサル/講座 → メディア出演
```

### フリーランス → プロダクト

```
受託案件 → ノウハウ蓄積 → テンプレート販売 → SaaSツール開発
```

### コンサル → 教育

```
スポットコンサル → パッケージ化 → オンライン講座 → 有料コミュニティ → 認定制度
```

## 分析時の注意

- 派生ビジネスは「今すぐやるもの」ではなく「スキルが積み上がった後の展望」として提示する
- ユーザーの現在のスキルレベルと、副業で獲得するスキルの差分を明確にする
- Tier 3は「可能性」であり「計画」ではない。過度に楽観的な描写を避ける
- 副業の本質（趣味のようか、地味な作業か）がTier展開の方向性に影響することを考慮する
