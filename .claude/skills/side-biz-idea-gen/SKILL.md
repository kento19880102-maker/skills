# 副業アイデア生成

ユーザーの状況に最適な副業アイデアを3〜5個提案する。4条件プレフィルタを適用し、競争力のあるアイデアのみを提示する。

## 入力

ユーザーのプロファイル（以下の情報の一部または全部）:
- 基本: スキル・経験、興味・関心、可処分時間、初期予算、リスク許容度
- 深掘り: 有料/無料サービス一覧、不満・困りごと、周囲の不満、願望・変化、ユニーク特性

$ARGUMENTS で親スキルからプロファイルが渡された場合はそれを使用する。情報が不足している場合は、最低限「スキル」と「可処分時間」を確認してから提案する。

## 3つの生成メソッド

以下の3つのメソッドを**この順番で**適用する。詳細は [generation-methods.md](references/generation-methods.md) を参照。

### メソッド1: サービス支出分析法（最優先）

ユーザーの有料/無料サービス利用パターンからアイデアを逆算する。

- **有料→無料**: 有料サービスの無料代替を発見・コンテンツ化・DB化
- **無料→収益化**: 無料サービスの利用パターンから収益化のヒントを見つける
- **不満→解決**: 不満・困りごとをサービス化する

### メソッド2: ユニーク交差点法

ユーザーのユニーク特性（属性の掛け算）からニッチを発見する。

- ユーザーの属性を3-5個リストアップ
- 2-3個の属性を掛け合わせ、「この組み合わせは自分だけ」を見つける
- その交差点でしか提供できない価値を考える

### メソッド3: カテゴリブレスト法（フォールバック）

8カテゴリから横断的にアイデアを出す。メソッド1-2で十分な候補が出なかった場合に使用。

- デジタルプロダクト / フリーランス・受託 / コンテンツ制作 / EC・物販
- コンサルティング・コーチング / SaaS・アプリ開発 / 教育・スキルシェア / 投資・資産運用

カテゴリごとの詳細は [idea-categories.md](references/idea-categories.md) を参照。

## 4条件プレフィルタ

アイデアをユーザーに提示する**前に**、以下の4条件で自己評価する。スコアリング基準の詳細は [generation-methods.md](references/generation-methods.md) を参照。

| 条件 | スコア | 判定基準 |
|------|--------|----------|
| 希少性（差別化） | ◎/○/△/× | 同じことをやっている人がいるか？ |
| ニーズ（欲求） | ◎/○/△/× | 人々が実際に求めているか？ |
| 潜在性（未発見） | ◎/○/△/× | 他が気づいていないギャップか？ |
| 参入障壁 | ◎/○/△/× | 後発が追いつけない要素があるか？ |

- **4条件中2つ以上が○以上のアイデアのみ提示する**
- 0-1個しか○がないアイデアは「検討したが除外」として簡潔にリストする

## ユーザー制約条件チェック

アイデア提示前に、ユーザーの制約条件を明示的に確認する:

- [ ] 初期予算内か？（0円制約がある場合、有料ツール必須のアイデアは除外）
- [ ] 週の可処分時間内で可能か？
- [ ] リスク許容度に合っているか？
- [ ] ユーザーが求める「趣味のような副業」に合っているか？（該当する場合）

## 各アイデアの分析フレームワーク

| 評価軸 | 内容 |
|--------|------|
| **概要** | 1-2文でアイデアの説明 |
| **生成メソッド** | どのメソッドから生まれたか（支出分析/交差点/カテゴリ） |
| **4条件スコア** | 希少性/ニーズ/潜在性/参入障壁 の ◎/○/△/× |
| **マッチ度** | ユーザーのスキル・興味との適合度（★1-5） |
| **市場ポテンシャル** | 需要の大きさと競合状況 |
| **初期投資** | 必要な資金と内訳の概算 |
| **初収益までの期間** | 現実的な目安（例: 1-3ヶ月） |
| **月収レンジ** | 軌道に乗った場合の現実的な月収幅 |
| **スケーラビリティ** | 時間の切り売りか、仕組み化可能か |
| **必要時間** | 週あたりの最低稼働時間 |
| **リスク** | 主なリスク要因 |
| **最初の一歩** | 今日・今週中にできる具体的アクション |

## 提案時の注意

- 「確実に稼げる」等の断定的表現は避け、現実的な期待値を伝える
- 投資に関しては情報提供のみとし、具体的な投資判断は行わない
- ユーザーのスキルレベルに合わない高難度アイデアは避けるか、段階的なステップを示す
- **前ラウンドで却下されたアイデアを再提案しない**（$ARGUMENTSに却下リストがある場合）
- 4条件プレフィルタの結果を正直に開示する。弱い条件を隠さない
