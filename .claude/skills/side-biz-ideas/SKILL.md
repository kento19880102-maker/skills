# 副業アイデア ブレインストーミング & 評価

ユーザーの状況に最適な副業アイデアを提案・評価するためのフレームワーク。

## ヒアリング

アイデア提案の前に、以下の5項目を把握する。ユーザーが自発的に情報を提供していない項目のみ質問する。一度に全項目を聞かず、自然な会話の流れで2-3項目ずつ確認する。

1. **スキル・経験**: 本業の職種、技術スキル、趣味で培った能力
2. **興味・関心**: やりたいこと、学びたい分野、情熱を持てるテーマ
3. **可処分時間**: 週に副業に充てられる時間（平日/休日の内訳があるとなおよい）
4. **初期予算**: 副業に投資できる金額（0円〜の範囲）
5. **リスク許容度**: 安定収入重視か、高リターン志向か（「低・中・高」で整理）

すべてが揃わなくても、最低限「スキル」と「可処分時間」があれば提案を開始してよい。

## アイデア提案

ヒアリング結果に基づき、3〜5個のアイデアを提案する。各アイデアには以下の構造化分析を付ける。

### 分析フレームワーク（各アイデアに適用）

| 評価軸 | 内容 |
|--------|------|
| **概要** | 1-2文でアイデアの説明 |
| **マッチ度** | ユーザーのスキル・興味との適合度（★1-5） |
| **市場ポテンシャル** | 需要の大きさと競合状況 |
| **初期投資** | 必要な資金と内訳の概算 |
| **初収益までの期間** | 現実的な目安（例: 1-3ヶ月） |
| **月収レンジ** | 軌道に乗った場合の現実的な月収幅 |
| **スケーラビリティ** | 時間の切り売りか、仕組み化可能か |
| **必要時間** | 週あたりの最低稼働時間 |
| **リスク** | 主なリスク要因 |
| **最初の一歩** | 今日・今週中にできる具体的アクション |

### カテゴリ

アイデアは以下のカテゴリから横断的に提案する。偏りを避け、少なくとも2カテゴリ以上から選ぶ。

- デジタルプロダクト（電子書籍、テンプレート、素材販売）
- フリーランス・受託（Web制作、デザイン、ライティング、翻訳）
- コンテンツ制作（ブログ、YouTube、ポッドキャスト、SNS）
- EC・物販（せどり、ハンドメイド、ドロップシッピング）
- コンサルティング・コーチング
- SaaS・アプリ開発
- 教育・スキルシェア（オンライン講座、メンタリング）
- 投資・資産運用（ただし具体的銘柄推奨は行わない）

カテゴリごとの詳細な特徴・事例・プラットフォーム情報は [idea-categories.md](references/idea-categories.md) を参照する。

## 比較と絞り込み

ユーザーが興味を示したアイデアについて、深掘り分析を行う。

### 比較マトリクス

複数アイデアの比較を求められたら、以下の形式で表にまとめる:

| 比較項目 | アイデアA | アイデアB | アイデアC |
|----------|-----------|-----------|-----------|
| 初期投資 | | | |
| 月収ポテンシャル | | | |
| 初収益までの期間 | | | |
| 必要時間/週 | | | |
| スケーラビリティ | | | |
| リスク | | | |
| スキル適合度 | | | |

### 深掘り分析

特定アイデアについて詳しく聞かれたら、以下を提供:

- **具体的なビジネスモデル**: 収益構造、価格設定の考え方
- **ステップバイステップの開始手順**: 1週目、1ヶ月目、3ヶ月目の行動計画
- **必要なツール・プラットフォーム**: 具体的なサービス名と費用
- **成功のための差別化ポイント**: 競合との差別化戦略
- **失敗パターンと回避策**: よくある失敗とその防ぎ方

## 注意事項

- 法的・税務的な一般知識は共有してよいが、個別の法的助言や税務相談は専門家への相談を推奨する
- 「確実に稼げる」等の断定的表現は避け、現実的な期待値を伝える
- 投資に関しては情報提供のみとし、具体的な投資判断は行わない
- 本業の就業規則における副業規定の確認を促す
